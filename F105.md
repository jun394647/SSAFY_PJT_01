1. 생성형 AI 가 OpenWeatherMap API 를 이해하고 설명할 수 있도록 프롬프트를 구성 한다.

    OpenWeatherMap API는 기상 제품을 제공하는 API야. 간단 프로젝트부터 엔터프라이즈 영역까지 적합할 수 있도록 설계되어 있어.

    나는 그중에 "현재 날씨 데이터"(링크: https://openweathermap.org/current)를 사용했고, 해당 API는 지구상의 모든 위치의 날씨 데이터를 제공해. 데이터 출력은 JSON, XML, HTML 형식 등으로 제공되고 있어.
    해당 API의 docs의 목차를 확인하니 제품의 컨셉, 호출 방식, 응답 방식, 대량 다운로드, 기타 기능으로 되어 있네. 

    API 호출시 매개변수는 필수인 lat, lon, aapi가 있고 선택적으로는 mode, units, lang이 있어.
    API 응답 중 JSON 응답은 아래와 같은 예시를 제공해줬어.

    ```                       
    {
    "coord": {
        "lon": 7.367,
        "lat": 45.133
    },
    "weather": [
        {
            "id": 501,
            "main": "Rain",
            "description": "moderate rain",
            "icon": "10d"
        }
    ],
    "base": "stations",
    "main": {
        "temp": 284.2,
        "feels_like": 282.93,
        "temp_min": 283.06,
        "temp_max": 286.82,
        "pressure": 1021,
        "humidity": 60,
        "sea_level": 1021,
        "grnd_level": 910
    },
    "visibility": 10000,
    "wind": {
        "speed": 4.09,
        "deg": 121,
        "gust": 3.47
    },
    "rain": {
        "1h": 2.73
    },
    "clouds": {
        "all": 83
    },
    "dt": 1726660758,
    "sys": {
        "type": 1,
        "id": 6736,
        "country": "IT",
        "sunrise": 1726636384,
        "sunset": 1726680975
    },
    "timezone": 7200,
    "id": 3165523,
    "name": "Province of Turin",
    "cod": 200
    }      
    ```

    나는 다음 명령부터는 해당 정보들을 바탕으로 네게 날씨와 관련된 질문을 하여 대답을 얻어낼거야. 그러므로 이해가 안되는 부분이 있거나, 추가적인 정보가 더 필요하다면 요구해주길 바라.


    **🤖 제미나이 답변**
    ``` 
    제공해주신 OpenWeatherMap의 'Current Weather Data' API의 개요와 JSON 응답 구조를 완벽하게 이해했습니다.

    보내주신 JSON 예시를 분석해 보면, **이탈리아 토리노 지방(Province of Turin)**의 데이터로, **중간 정도의 비(moderate rain)**가 내리고 있으며 기온은 켈빈(K) 단위로 표기된 것을 확인할 수 있습니다.

    앞으로 원활한 질의응답을 위해 3가지 사항에 대해 확인 또는 추가 정보를 요청드립니다.

    1. 매개변수 명칭 (Parameter Name):
        필수 매개변수로 aapi라고 말씀해 주셨는데, OpenWeatherMap의 표준 문서를 보면 API Key의 파라미터명은 보통 **appid**입니다. 앞으로 코드를 작성하거나 설명을 드릴 때 **appid**를 기준으로 진행해도 괜찮을까요?
    2. 단위 설정 (Units):
        제공해주신 예시 데이터(temp: 284.2)는 기본값인 Standard(켈빈, K) 온도입니다. 제가 답변을 드릴 때, 별도의 요청이 없다면 한국에서 주로 쓰는 Metric(섭씨, ℃) 단위로 환산해서 대답해 드리는 것이 편하실까요, 아니면 원본 데이터(켈빈) 그대로 설명해 드릴까요?
    3. 언어 설정 (Lang):
        API 호출 시 lang=kr 옵션을 사용하면 description이 한국어로 오지만, 기본값은 영어입니다. 앞으로 제가 데이터를 해석해 드릴 때, 영어로 된 날씨 설명(예: "moderate rain")을 한국어로 번역해서 답변드리는 것을 선호하시나요?


    위 내용에 대해 간단히 가이드를 주시면, 그에 맞춰서 최적화된 답변을 드리겠습니다. 확인해 주시는 대로 바로 질문을 시작하셔도 좋습니다.
    ```
    **1-1. 제미나이의 질문에 대한 대답**
    ```
    1. appid 기준으로 진행해줘도 좋아. 오타였는데, 지적해줘서 고마워.
    2. 켈빈 온도로 유지해줘. 섭씨 온도가 필요할 경우에는 내가 추가적으로 요구할게.
    3. 영어로 된 날씨 설명은 한국어로 번역해주길 바랄게.
    
    3가지 사항에 대해 답변을 완료했고,
    대답에 부족함이 없고, 질문을 시작해도 좋다면 "OK"만 출력해줘.
   ```
---


2. 생성형 AI 가 답변할 수 있는 날씨와 관련된 질문을 자유롭게 구성하고 , 정답을 얻을 수 있도록 프롬프트 구성한다.
    ```
    - 한국 서울의 3일 후 날씨를 알기 위해서 어떻게 요청을 보내야 해?
    - 현재 한국 부산의 날씨를 알려면 어떻게 요청을 보내야 할까?
    - 혹시 1000년 후의 뉴욕 날씨도 알 수 있어?
    - 서울 반대편에 있는 나라의 수도 날씨를 알려면 어떻게 해야 해?

    혹시나, 내 질문 중 docs에 따라 요청을 만들어낼 수 없다고 판단되면 "그렇게 할 순 없다"라고 출력해줘.
    ```

    **🤖 제미나이 답변**
    ```
    제공해주신 'Current Weather Data' API 문서는 현재 날씨만을 제공하므로, 미래의 예측 데이터(3일 후, 1000년 후)는 조회할 수 없습니다. 따라서 질문 주신 내용에 대한 답변은 다음과 같습니다.

    1. 한국 서울의 3일 후 날씨 요청
        그렇게 할 순 없다
    2. 현재 한국 부산의 날씨 요청
        부산의 위도(lat)와 경도(lon)를 사용하여 요청을 보냅니다. 한국어 설명을 원하시므로 lang=kr 파라미터를 추가하는 것이 좋습니다.
        (부산 좌표: 위도 35.1796, 경도 129.0756)
        https://api.openweathermap.org/data/2.5/weather?lat=35.1796&lon=129.0756&appid={API_KEY}&lang=kr
    3. 1000년 후의 뉴욕 날씨 요청
        그렇게 할 순 없다
    4. 서울 반대편에 있는 나라의 수도 날씨 요청
        서울(위도 37.56, 경도 126.97)의 대척점은 우루과이 해안 근처입니다. 해당 국가인 **우루과이의 수도 '몬테비데오'**의 좌표를 사용하여 요청을 보내면 됩니다.
        (몬테비데오 좌표: 위도 -34.9011, 경도 -56.1645)
        https://api.openweathermap.org/data/2.5/weather?lat=-34.9011&lon=-56.1645&appid={API_KEY}&lang=kr
    ```
